The HI catalogue and the continuum catalogue are generated independently in the first instance. They contain some common columns that can be used to identify plausible counterparts. the quantities in common are:
 
 
MHI - HI mass
MH - dark halo mass
x_coo  - euclidean coordinates
y_coo
latitude - spherical coordinates
longitude
redshift
inclination
q - axis ratio
PA
Optclass – 2 for spiral or 1 for elliptical
size (although size in HI and size in continuum not exactly the same – we migh want to generate a predicted HI size from the continuum size) 
 
 
The quantities not in bold do not need to be matched: once a match is found, one set of variables can be overwritten with the other. Coordinates and PA are random and uncorrelated so there is no harm in changing them. Redshift is of course important but the catalogues could be cross-matched redshift bin by redshift bin, thus avoiding the problem. inclination, optical class and axis ratio are dependent quantities (spirals and ellipticals start with a different instrinsic axis ratio, which is converted to q using inclination). Finally, for AGN (identified by having Radioclass>3 in the continuum catalogue) we don’t expect a particular correlation between the radio continuum size and morphology and the HI one.
 
 
Not all objects need to have a counterpart, and not all common tags need to be correlated. At the moment I am only using MHI to find association, but there is a big scatter in the correlation of the other quantities. The todo list for this are:
